# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 사용자가 자신을 표현하는 다양한 소셜 미디어, 포트폴리오, 외부 작업물 등의 링크를 하나의 웹 페이지로 모아서 쉽게 공유할 수 있는 통합 링크 프로필 서비스 구축.
- **대상 사용자**: 크리에이터, 인플루언서, 프리랜서, 포트폴리오 공유가 필요한 취업 준비생 등 여러 웹 경로를 하나로 묶어 홍보하고자 하는 모든 개인.
- **기술 및 디자인 원칙**: 초기 MVP 배포는 핵심 뼈대 기능에 집중하는 경량화된 형태로 개발됩니다. 프론트엔드 UI/UX 디자인은 범용성이 높은 **`shadcn/ui`** 디자인 시스템 컴포넌트를 베이스로 구축하여 추후 테마 확장을 용이하게 합니다.

## 2. 핵심 기능 목록

### 필수 기능 (Must-have - 배포 목표)
- **회원가입 및 로그인 (Authentication)**: 사용자를 식별하기 위한 계정 관리 (가입 시 구글 Gmail 계정의 `@` 앞부분을 추출하여 `displayName` 최초값으로 자동 세팅)
- **고유 프로필 URL 발급 (Profile URL)**: `mylink.com/displayName` 형태의 닉네임 기반 고유 접속 링크 생성 (`displayName` 중복 검사 포함)
- **링크 통합 관리 기능 (Link Management)**: 
  - 링크 블록 생성 (단순 텍스트 제목 + 목적지 URL 입력)
  - **링크별 아이콘 노출 지원** (각 링크 대상 URL의 파비콘을 구글 API(Google Favicon API)를 활용하여 자동 연동)
  - 등록된 링크 본문 수정 (**인라인 편집** 방식) 및 삭제
- **프로필 설정 (Profile Customization)**: 
  - 닉네임 및 짧은 자기소개글(Bio, 최대 80자 제한) 노출 및 **인라인 편집(In-line Editing) 방식을 통한 직접 수정 기능** 지원
- **프로필 페이지 뷰어 (Viewer)**: 방문자가 접속했을 때, 모바일에 최적화된 반응형 UI로 링크 목록을 렌더링
- **실시간 미리보기 화면 (Live Preview)**: 관리자가 대시보드에서 수정하는 내용이 우측의 '스마트폰 화면 프로토타입' 영역에 실시간 반영

### 선택 기능 및 향후 개선 스펙 (Post-MVP / v2.0)
*※ 초기 배포 환경(MVP)의 빠른 런칭을 위해 보류된 기능이며, 이후 V2 스펙이나 프리미엄 요건으로 고도화될 목록입니다.*

- **[제안 1] 방문자 통계 및 애널리틱스 (Analytics)**: 전체 방문자 수(PV), 각 링크별 클릭 수(CTR) 등의 대시보드 통계 제공 (크리에이터들에게 필수 동기 부여 요소)
- **[제안 2] 다채로운 테마 및 스타일링 세분화**: 
  - 기본 제공될 `shadcn/ui` 베이스 템플릿 외에, 배경색, 블록 모서리 마감(Radius), 폰트 스타일 등을 개별 커스터마이징 가능
- **[제안 3] 소셜 로그인 연동 (OAuth 2.0)**: 구글(Google), 카카오, 깃허브 등 기존 소셜 계정을 활용한 간편 가입 (사용자 이탈률 감소 효과)
- **[제안 4] 리치 미디어 및 특수 블록 지원**: 단순히 텍스트 형태의 링크뿐만 아니라 하단에 작게 고정되는 '소셜 미디어 전용 아이콘 바(인스타그램, 트위터 아이콘 등)'나 유튜브 영상 '임베드 블록' 제공

## 3. 기능 상세 설명

### 3.1. 인증 및 계정 관리
- 신규 사용자 가입 시 입력된 이메일(구글 Gmail 기준) 주소에서 `@` 앞부분의 아이디를 추출하여 **`displayName`의 기본값**으로 자동 설정합니다.
- 해당 `displayName`은 사용자 편의에 따라 자유롭게 변경 가능하지만, 프로필 접속 URL 슬러그(slug)로 함께 작동하기 때문에 변경 시 기존 사용자들과의 중복 여부를 실시간으로 검사해야 합니다.

### 3.2. 링크 및 프로필 관리 대시보드 (Admin Dashboard)
- **내부 구조**: 좌측은 설정/입력 컨트롤 영역, 우측은 실시간 모바일 미리보기(Preview) 영역으로 나누어 직관적인 피드백을 제공합니다.
- **인라인 편집 지원**: 닉네임, 소개글, 그리고 각 링크의 텍스트와 URL 등 요소들은 별도의 입력 폼이나 모달 이동 없이 대시보드상에서 텍스트를 직접 클릭해 수정하는 **인라인 편집(In-line Editing)** 방식으로 설계합니다.
- **링크 편집기**: 링크는 단순 리스트 아이템 형태로 세로형 배치됩니다. (별도의 드래그 앤 드롭 순서 변경이나 활성화/비활성화 토글 기능은 제공하지 않음)

### 3.3. 프로필 퍼블릭 페이지
- 로그인하지 않은 외부 방문자가 URL로 접근 시 보게 되는 화면입니다. 
- 오로지 정보 전달에 집중된 모바일 친화적인 Single Page 구조를 갖추며, 화면 최상단에는 **닉네임과 소개글**이 위치하고 그 아래로 파비콘이 포함된 링크 블록 버튼들이 나열됩니다. (별도의 프로필 이미지 영역 없음)

## 4. 데이터베이스 모델링 (MVP 기준 - NoSQL 서브컬렉션 초안)
서브컬렉션(Sub-collection)을 활용하는 NoSQL(Firestore 등) 구조에 맞춘 MVP 스키마입니다. 별도의 조인(Join) 없이 프로필 페이지 렌더링을 최적화하기 위해 사용자 정보와 프로필 정보를 통합하고, 링크 데이터만 서브컬렉션으로 분리합니다.

```text
📦 Collection: users
 ┗ 📜 Document: {userId}
    ┣ 🏷️ email        : string    (고유 이메일)
    ┣ 🏷️ passwordHash : string    (암호화된 비밀번호 - 소셜 로그인 도입 시 선택)
    ┣ 🏷️ displayName  : string    (화면 닉네임 겸 고유 URL 슬러그. 구글 메일 앞부분 문자열로 초기 세팅됨)
    ┣ 🏷️ bio          : string    (짧은 자기소개 텍스트)
    ┣ 🕒 createdAt    : timestamp (가입 시각)
    ┃
    ┗ 📦 Sub-Collection: links
       ┗ 📜 Document: {linkId}
          ┣ 🏷️ title      : string    (입력된 링크 제목)
          ┣ 🏷️ url        : string    (목적지 URL)
          ┣ 🏷️ faviconUrl : string    (구글 Favicon API를 통해 자동 획득한 캐싱 경로)
          ┗ 🕒 createdAt  : timestamp (순차적 단순 정렬을 위한 등록 시각)
```
- **설계 포인트**: 기존의 역할 분리형 `PROFILES` 테이블 대신 `users` 단일 도큐먼트 안에 `displayName`, `bio` 등을 병합해 읽기 비용을 낮추었습니다.
- **링크 정렬 방식**: `links` 컬렉션 구조에는 별도의 상태값(토글)이나 인덱스(순서)가 없으므로 `createdAt` 기준으로 쿼리하여 리스팅합니다.
