# 마이링크 (MyLink) - 사용자 시나리오 (User Scenario)

본 문서는 명확한 흐름 파악을 위해 **"사용자는 [목적]하기 위해 [행동]한다"** 형식으로 기재되었습니다.

---

## 1. 방문자 (Visitor): 마이링크 페이지 방문 및 이용 과정

### 1-1. 정상적인 프로필 탐색 로직
1. 방문자는 특정 소유자의 링크 모음을 **확인하기 위해** 브라우저 주소창(또는 타 SNS의 프로필 링크)에 `mylink.com/displayName` URL로 **접속한다.**
2. 방문자는 이 프로필이 누구의 것인지 **파악하기 위해** 화면 상단의 닉네임과 짧은 소개글(Bio)을 **읽어 본다.**
3. 방문자는 자신의 관심사와 일치하는 웹사이트로 **이동하기 위해** 원하는 파비콘(아이콘)과 제목이 적혀 있는 링크 블록 버튼 중 하나를 **클릭(터치)한다.**
4. 방문자는 클릭한 링크 탭을 통해 원본 웹페이지 컨텐츠를 **소비하기 위해** 이동된 창에서 화면을 **살펴본다.**

### [AI 제안] 1-2. 예외 상황 처리 흐름 (404 에러 대응)
1. 방문자는 특정 페이지에 **접속하기 위해** URL을 입력했으나 주소의 오타 등으로 인해 이탈하는 것을 **방지하기 위해** 시스템은 "존재하지 않는 페이지입니다" 안내 화면과 함께 `마이링크 홈으로 가기` 버튼을 **제공하여 클릭을 유도한다.**

---

## 2. 소유자 (Owner): 링크 추가/수정/삭제 관리 과정

### 2-1. 새로운 링크 블록 추가 (Create)
1. 소유자는 자신의 새로운 웹사이트(목적지)를 프로필에 **추가하기 위해** 대시보드 화면에 위치한 `새 링크 추가` 버튼을 **클릭한다.**
2. 소유자는 빈 텍스트 형태로 생성된 블록에 내용을 **채우기 위해** 링크 제목과 목적지 URL을 각각 **입력한다.** (이때 URL 입력 시 `http` 프로토콜이 누락된 채 주소만 입력하더라도, 사용자 편의를 위해 시스템은 자동으로 `https://`를 맨 앞에 삽입해 준다.)
3. 소유자는 입력한 웹사이트가 정상적으로 처리되었는지 시각적으로 **확인하기 위해** 구글 API에 의해 자동으로 연동되어 뜨는 '파비콘 이미지'가 노출되는지 **점검한다.**

### 2-2. 기존 링크의 인라인 수정 (Update)
1. 소유자는 기존에 등록해 둔 링크의 제목이나 URL을 좀 더 직관적으로 **바꾸기 위해** 대시보드 리스트의 텍스트 영역을 즉시 마우스로 **클릭(포커스)한다.**
2. 소유자는 새로운 텍스트로 변경사항을 **저장하기 위해** 키보드로 타이핑한 후 화면의 외부 빈 영역을 클릭하거나 `Enter` 키를 **입력한다.** (단, 포커스 상태에서 텍스트를 완전히 다 지우고 엔터를 치는 등 빈칸(Empty)으로 변경 시도 시, 빈 문자열 삽입을 방지하고 "수정 전 기본 텍스트"로 즉시 무음 롤백(복구) 처리된다.)
3. 소유자는 본인이 수정한 과정이 방문자 페이지에 올바르게 반영될지 **점검하기 위해** 화면 우측에 위치한 '실시간 스마트폰 렌더링 미리보기 창'을 **살펴본다.**

### 2-3. 기존 링크 영구 삭제 (Delete)
1. 소유자는 더 이상 쓸모 없어진 목적지 URL을 자신의 프로필 목록에서 **제외하기 위해** 해당 링크 블록 우측 끝에 있는 `삭제(휴지통)` 아이콘 버튼을 **클릭한다.**
2. 소유자는 여러 개의 링크를 빠르고 직관적으로 치우는 정리 작업을 **수행하기 위해** 재차 묻는 번거로운 팝업 창(Confirm) 없이 누른 즉시 해당 링크 블록이 영구 삭제되는 것을 **확인한다.**
